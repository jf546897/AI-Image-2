# 贡献指南

感谢你为 **AI Image 2 Portable** 提交改进。

本仓库当前以**便携运行、配置可用、提供方兼容性清晰**为优先目标。为了降低维护成本，请尽量保持提交小而聚焦。

## 1. 贡献前先确认

在开始前，请先判断你的改动属于哪一类：

- **Bug 修复**：现有行为与预期不符，且可以复现。
- **功能建议**：希望新增能力、优化流程或改进体验。
- **Provider 兼容性反馈**：某个 OpenAI 风格 API / 网关是否可用，或哪些端点存在差异。
- **文档维护**：说明、模板、流程文档更新。

请优先使用对应的 Issue 模板，先把问题边界说清楚，再开工修改。

## 2. 提交原则

### 2.1 改动要聚焦

- 一个 PR 尽量只解决一个问题。
- 不顺手重构无关代码，不混入大范围格式化。
- 不提交 `cache/`、日志、临时文件、个人配置或打包产物。

### 2.2 敏感信息不能入库

严禁提交以下内容：

- `local_config.json`
- 真实 API Key / Token / Cookie
- 带敏感头信息的完整请求日志
- 任何包含私有服务地址且不适合公开的信息

如果需要提供日志，请先脱敏。

### 2.3 中文优先

为便于维护，新增文档、注释、Issue/PR 描述建议优先使用中文；必要时可补充英文关键字。

## 3. 开发与验证建议

如果你的改动涉及运行逻辑，请至少完成与你改动直接相关的验证：

### 3.1 基础启动验证

1. 双击 `Start_AI_Image2.bat`
2. 确认浏览器能打开 `http://127.0.0.1:8012/`
3. 双击 `Stop_AI_Image2.bat`
4. 确认服务可以正常停止

### 3.2 配置验证

- 填写 `API URL Base`
- 填写 `API Key`
- 点击保存，确认没有报错

### 3.3 图片生成验证

如果你有可用的测试提供方：

- 选择或填写可用模型
- 提交一次最小化图片生成请求
- 记录成功或失败现象

如果你没有可用的测试密钥，请在 PR 中明确写出**未验证部分**。

## 4. Provider 兼容性反馈要求

如果你反馈或修复的是提供方兼容性问题，请尽量附带：

- 提供方名称
- Base URL（脱敏）
- 成功模型与失败模型
- `/models` 是否可用
- 图片生成请求走哪个端点
- 关键错误信息或 HTTP 状态码

这样比只说“某家不兼容”更有价值，也更利于后续沉淀兼容矩阵。

## 5. PR 提交流程

提交 PR 时请：

1. 填写 PR 模板；
2. 写清楚改动目的、验证方式、剩余风险；
3. 如果有用户可见变化，同步更新相关文档；
4. 如果是阶段性变更，请说明后续计划，不要伪装成已完成。

## 6. 不建议提交的内容

以下内容通常不适合直接合入：

- 为未来可能需求提前加的大型抽象
- 没有复现依据的“猜测式修复”
- 把多个独立问题打包在一个 PR 中
- 未说明来源的大段第三方代码

## 7. 行为准则

讨论问题时请直接、具体、基于证据：

- 说清复现条件；
- 区分“已验证”和“推测”；
- 尽量给最小复现路径；
- 尊重不同使用环境，不把个别网关行为当作通用标准。
