# 路线图

本文档用于说明 **AI Image 2 Portable** 的维护方向，帮助贡献者判断哪些改动更容易被接受。

## 当前优先级

### P0：基础可用性

- 启动 / 停止流程稳定
- 本地配置保存可靠
- 关键错误提示可读
- 便携目录在新机器上可直接运行

### P1：Provider 兼容性

- 明确不同 OpenAI 风格接口的兼容边界
- 沉淀可复现的兼容性反馈模板
- 区分“配置错误”和“提供方不兼容”

### P2：维护体验

- 补齐社区协作模板
- 完善贡献、安全与变更记录文档
- 降低新贡献者上手成本

## 近期可接受方向

以下方向通常值得优先投入：

1. **兼容性证据化**
   - 用最小复现步骤描述某家提供方能否工作
   - 记录模型列表、图片生成、报错行为的差异

2. **错误反馈更明确**
   - 用户能更快知道是密钥问题、端点问题，还是模型不支持

3. **便携运行更稳**
   - 在 bundled runtime 与本机 Python 两种路径下都更容易定位问题

4. **文档闭环**
   - 文档能覆盖启动、配置、提问、报错反馈与贡献流程

## 暂不优先

以下事项不是完全拒绝，但当前通常不会排在前面：

- 为未知未来场景提前做大规模架构重写
- 与当前便携目标无关的复杂插件系统
- 没有验证依据的“通用兼容层”
- 仅为美观而不提升可维护性的改造

## 进入实施前建议

如果你准备提交较大改动，建议先在 Issue 中说明：

- 要解决的具体问题
- 为什么现有方案不够
- 你的最小实现边界
- 如何验证改动确实有效

这样可以减少返工，也能降低与其他贡献者的冲突概率。
